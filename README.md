**Amazon Sales Performance Analysis — Excel Case Study**

**1. Introduction**

E-commerce sellers rely on data-driven decision-making to optimize sales, manage inventory, and improve customer engagement. This case study focuses on analyzing Amazon sales data using Microsoft Excel. The objective is to extract insights related to product performance, revenue trends, customer behavior, and operational efficiency.
The entire project is executed using Excel, covering data loading, cleaning, preprocessing, analysis through PivotTables, and dashboard creation.

**2. Business Problem Statement**

An Amazon seller shared one month of transactional sales data and requires structured insights to answer key business questions:

Which products generate the highest total sales?

Which categories perform best?

Which cities show strong or weak performance?

How do payment methods influence orders?

What percentage of orders are completed, cancelled, or pending?

The goal of this analysis is to build a complete Excel dashboard that provides clear, actionable insights to support strategic decision-making.

**3. Project Objectives**

The analysis aims to:

Load and understand the dataset structure

Clean and preprocess the data to ensure accuracy

Create calculated fields for deeper understanding

Perform descriptive and comparative analysis using PivotTables

Develop an interactive dashboard highlighting key KPIs

Present insights and recommendations based on findings

**4. Methodology**

4.1 Data Loading

Imported data into Excel

Verified column structure

Checked data types and formatting


Identified missing values

**4.2 Data Cleaning**

Steps performed:

Removed duplicates

Standardized formats

Converted Date → Date format

Converted Price, Quantity, Total Sales → Number format

Cleaned text fields using TRIM and PROPER

Fixed inconsistent category and location names

Validated totals (Total Sales = Price × Quantity)

**4.3 Data Preprocessing**

Calculated fields created:

Month

Day

Year

Order Status grouping

Since revenue already exists as Total Sales, no additional revenue calculation was required.

**4.4 Data Analysis Using PivotTables**

PivotTables were created to analyze:

Category-wise total sales

Product-wise performance

City-wise total sales

Payment method usage

Order status distribution

Top customers by total spending

**4.5 Dashboard Development**

Dashboard Includes:

Total Sales by Category

Total Sales by Product

Sales by Customer Location

Order Status Breakdown

Payment Method Distribution

Top Customers Analysis

**Filters (Slicers):**

Category

Customer Location

Order Status

Month

**5. Conclusion**

This project demonstrates how Microsoft Excel can be used as a powerful data analysis and dashboarding tool. The interactive dashboard provides a clear overview of sales performance, customer behavior, and operational efficiency, supporting better business decision-making.
